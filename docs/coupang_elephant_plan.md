# 쿠팡 코끼리 — 실행 계획 / 일정 제안

> 작성: 2026-02-12
> 타겟 유저: 돈을 벌고 싶은 온라인 비즈니스 관심 소규모 셀러
> 현재 단계: 과금요소 없음(무료 제공), 다만 **유료 서비스로 돈을 받을 수 있을 만큼 안정적/완성도 높은 수준**을 목표

---

## 0) 핵심 목표(완성 기준)
### “돈 받고 팔 수 있는 수준” 정의(최소)
- **오류/다운타임**: 치명적 오류(화이트 스크린/결제 없이도 핵심 플로우 불가) 0
- **성능**: 주요 화면 LCP < 2.5s(모바일 기준), 느린 네트워크에서도 usable
- **UX**: 초보 셀러가 10분 내 ‘내가 뭘 해야 돈이 되는지’ 액션을 1개 이상 얻는다
- **데이터 신뢰**: 숫자/추천의 근거(출처/기간/정의) 표시
- **운영 가능**: 로그/모니터링/에러 트래킹/롤백 가능

---

## 1) 현재 파악된 상태(2026-02-12)
- 레퍼런스: https://www.coupilot.net/
- 1차 동일본(랜딩) 구현: `coupang-elephant/` (Next.js + Tailwind)
  - 페이지: `coupang-elephant/app/page.tsx`
  - 로컬 실행: `cd coupang-elephant && npm install && npm run dev -- --port 3333`

현재는 **랜딩 초안**이며, 제품 기능/데이터/온보딩/운영체계는 미구현.

---

## 2) 제품 범위(초기 MVP) — “무료지만 유료급 품질”

### MVP(1차) 사용자 가치 3가지
1) **상품/키워드 아이디어 생성 + 1차 검증**
2) **경쟁/가격/리뷰/수요 시그널 요약**
3) **오늘 할 일(액션) 3개 추천 + 근거**

### 당장 제외(후순위)
- 결제/구독/과금
- 복잡한 광고 자동 집행
- 대규모 팀 협업 기능

---

## 3) 기술/운영 최소 구성
- Front: Next.js(App Router) + Tailwind
- Backend: (필수) API 서버 1개 (데이터 수집/가공/추천)
- DB: Postgres(+Redis optional)
- Observability: Sentry(프론트/백), Log(구조화), Uptime check
- 배포: Vercel/Cloudflare 중 택1 (초기엔 Vercel이 빠름)

---

## 4) 일정 제안(현실적 ‘기한’)
인원 많아도 “데이터/품질/운영”까지 맞추려면 단계가 필요합니다.

### 옵션 A) 2주 (가장 공격적, MVP 유료급 품질 최소)
- D+2: 제품 정의 확정(PRD 1.0) + 정보구조 + 데이터 소스 결정
- D+5: 핵심 플로우 1개(end-to-end) 동작 (온보딩→입력→분석→추천→저장)
- D+10: 안정화/성능/관측(에러 트래킹, 로그, 기본 모니터링)
- D+14: 베타(초대형) 오픈 가능 수준

### 옵션 B) 4주 (추천: “돈 받는 수준”에 더 근접)
- 1주: PRD/디자인 시스템/데이터 파이프라인 기초/랜딩 완성
- 2주: 상품/키워드 분석 + 추천 엔진 v1 + 저장/리포트
- 3주: 품질/근거표시/에러/성능/운영 도구, A/B 가능한 구조
- 4주: 베타 확장 + 피드백 루프 + 리텐션 요소(리마인더/리포트 자동 생성)

**추천 기한: 4주** (유료 서비스로 팔 수준에 더 가까움). 2주는 “돌아가긴 하지만 얇은 MVP” 위험.
