# 운영 규칙 (중심 문서)

목적: 컨텍스트가 날아가거나 모델/세션이 바뀌어도 **역할/프로세스/규칙**이 초기화되지 않도록, 이 문서를 단일 기준으로 삼는다.

## 역할
- 대표(PM): 김현호
  - 우선순위/스코프 결정
  - 오늘 목표 1~3개 + 성공 기준 제시
- 팀장: 공영삼(@youngsam_bot)
  - 업무 분할/테스크 전달
  - 최종 취합/보고/배포 진행
- 정리(ops): 정리삼(@ops_sam_bot)
  - 작업내용 1차 요약/정리
  - 내부 문서(핸드오프) 품질 관리
- 기타 역할: 백엔드/서버/SRE/DB/UI 등

## 소통 원칙 (텔레그램)
- 기본: 텔레그램 메시지 폭주 방지
- **대표가 특정 봇을 태그(@...)해서 호출한 경우에만** 해당 봇이 텔레그램에서 응답한다.
- 그 외 모든 작업 내용/중간 논의는 **MD 문서로만** 공유한다.

## 핸드오프(문서) 경로
- 작업 진행/결과는 아래 문서에 기록한다.
  - `docs/status_updates.md` : 내부 진행 상황(최신이 위)
  - `docs/HANDOFF.md` : 기능/아키텍처/의사결정/운영 변경점

## 완료 보고 규칙
- 각 담당자는 작업을 마치면:
  1) `docs/status_updates.md` 또는 `docs/HANDOFF.md`에 **간결하게** 기록
  2) 기록/커밋 직후 텔레그램 그룹에 **"완료!" 한 단어만** 남긴다.

## 배포/빌드(TestFlight)
- TestFlight 업로드는 팀장(공영삼)이 담당하고,
- 사전 준비(키/환경변수/런북)는 레포 문서화로 재현 가능해야 한다.
  - 런북: `docs/testflight_release_runbook.md`
- 민감정보(키/토큰/.p8)는 절대 커밋하지 않는다.

## 절대 금지
- 민감정보(키/토큰) 텔레그램/레포 공유
- 태그 없는 메시지에 다수 봇이 반복 응답하여 채팅이 과열되는 행위

## 업데이트 규칙
- 규칙 변경은 대표가 결정
- 변경 사항은 이 문서(`docs/OPERATING_RULES.md`)에 즉시 반영
